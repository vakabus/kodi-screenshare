# Requirements and design doc

Before doing anything else, make sure to read:

- docs/prd.md
- docs/tech-design.md

The PRD is the most autoritative source of requirements. Do not edit the PRD unless explicitely asked. The technical design comes second in priority. If these documents disagree, the PRD takes precedence.

When asked to implement something:
1. Consult the PRD. Does the request follow the requirements? Demand the user for PRD updates if the request doesn't match the requirements.
2. Consult the technical design. Does it need to be updated with new details? If it does, update it and ask for confirmation whether to continue with the implementation.
3. Only when the above 2 rules have been followed, proceed with the actual implementation.
